### 2. CUDA capability mismatch on Kaggle T4 / P100
Kaggle assigned a Tesla P100 (sm_60). The pip-installed PyTorch 2.10+cu128
requires sm_70+. PyTorch fell back silently to CPU for tensor ops (CUDA
device was detected but kernels are not compatible — model ran on `cuda` label
but dispatched to CPU). This means TTT step times would be slow on live runs.
**Fix:** Pin PyTorch to a version that supports P100:
  `pip install torch==2.0.1+cu117 --index-url https://download.pytorch.org/whl/cu117`
Or request a T4 (sm_75) explicitly in the kernel.
